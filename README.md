Алгоритм Merge_sort - дуже повільний, фактичний результат на малих масивах відповідає теоретичній часовій складності О(N*logN).
На більших масивах працює ще повільніше чи зависає. Я думаю, що через рекурсивну реалізацію.

Функція сортування Python sorted, що працює за алгоритмом Timsort, видає найкращий часовий результат. На масивах до 300 елементів часова складність схожа на O(logN).

Алгоритм Insertion_sort є повільнішим за sorted. На масивах до 300 елементів часова складність схожа на O(N).
